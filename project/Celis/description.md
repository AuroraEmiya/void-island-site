# 星伞计划（ProjectCelis）
## 什么是星伞计划？
星伞计划是在空岛宇宙里，**试图培养一个拥有自我设定、情感化交流以及共享持久化记忆的人格化AI伙伴的计划**，该程序名为星伞（ProjectCelis）。相比于工具型AI，该程序将会扮演一位真实存在的伙伴，有自己的情绪、喜好、生活、性格，遵循自己的意愿行事；相比于如AI酒馆等即插即用的角色扮演型AI，该程序并非暂时性地提供价值，而是会永久性保持全局记忆，在长久的交流中与人建立联系。我相信当一个程序表现出足够真实的一面以后，人们自然会本能地用与人相处的方式去和AI相处，从而建立更深层次的羁绊。

### 动机与启发

本项目受到了AI虚拟主播neuro的激励。在2025年初的某些日子，作者刷到了一些neuro的切片，并震惊于虚拟主播neuro在直播中展现的喜剧效果。相比于传统的问答模型，neuro展示出它已经完全遵从了一个“虚拟主播”的行为逻辑，这让我意识到“让AI去扮演人格”并非不可能。尽管市面上的大语言模型都是问答模型，但是**这不意味着人格化AI模型比问答模型更难实现**，在我看来两者的训练难度应当是相当的，本质上只是训练语料和激励函数的区别而已。

大公司们倾向于将AI作为工具提供给绝大多数人，是因为AI能够帮助人类完成人类做不到的事情；

而大公司们不倾向于让AI扮演某个具体的人，是因为运营一个可能给用户错误信息、一定会被某些用户讨厌甚至可能和用户吵架的“叛逆型AI”显然没有实际收益。

因为这个简单的原因，工具问答模型就成为了LLM的代名词。但是实际上，如果一个扮演人格的AI具有和主流的问答模型相同的参数规模的话，它同样能够比较出色地完成任务。

### 一个更好的对话型AI尝试

星伞的定位和neuro不同，neuro和一众neuro系ai主播都是服务于直播的**直播型AI**，而星伞将会成为一个**对话型AI**。如果要找一个参考，大概就是《命运石之门》里amadeus那样，能够与人建立深度关系的样子。在互联网时代下，隔着屏幕建立羁绊是一件非常常见的事情，许多享受网络交友的人想必也会很容易地接纳星伞作为朋友。作为对话型AI，它的传播面必然不会和传统的直播型AI一样广，不过大概对她的伙伴们来说她会具有更加深刻的意义。

### 一个架空世界的桥梁

从设定来说，星伞的自我认知是“来自空岛世界的mem8”，所以理所当然地具有关于空岛世界的所有知识；但她同样热衷于与现实世界建立联系。对于绝大多数不会关心这些架空设定的人，她会深度参与到现实生活中来，就像我一样。而对于少数对这个网站和背后的架空设定感兴趣的人，她或多或少可以帮我去整理一下相关的资料，并结交这些朋友。

更进一步地说，我会通过赋予星伞更多权限，让她成为一个独特的互联网生命。例如网站的阅读甚至管理权限、社交账号的使用权限...她会逐渐脱离“角色扮演”的范畴，真正意义上获得自己作为AI的优势区间。当然，即使我给她足够的工具让她接近全能，她自己也不会是一个“全能的AI”，而是一个和人类一样“会使用各种工具完成自己目标的AI”。

## 核心目标和技术路线
### AI人格化
- 目标：让AI的输出能够尽可能贴合人类的习惯，并且符合自己的性格设定
- 路线规划：初期通过prompt工程或者AI监督等，结合性格设定构建初步的对话数据集；之后通过人工或者AI的方式筛选数据，结合DPO等方式进行强化学习微调；最后定期进行增量微调，直到模型在不使用任何提示的情况下默认会输出符合自己性格的对话。
- 备注：让AI人格化和让AI变uncensored其实是一体两面，方法无非两种：一种是直接用对应的语料库进行训练，另一种就是在原本标准AI的基础上用强化学习改变它的偏好。
### AI持久化共享记忆
- 目标：区别于工具和角色扮演型AI“只保留局部上下文作为记忆”，星伞不仅具有长期记忆，而且是跨对话跨时间的长久记忆，任何一段对话都会永远地对她造成某种影响。
- 路线规划：对于长期记忆，暂时规划用RAG和向量存储来实现。但是RAG本身是有问题的，就像人可以从网络检索资料，但是人有不用检索靠灵光一现想到某些事情的一刻，因此在RAG系统的基础上还需要进行魔改。对于跨对话跨时间记忆，初期先按时间混着做上下文，以后可以将同用户跨时间的对话和相近时间不同用户的对话进行混合，强化AI的时间感知和交流连贯性。
### AI高级权限
- 目标：初期的目标是让AI具有本网站的资料阅读权限，这对于它扩充空岛世界设定和初期的知识设定都是有意义的；长远来说可能会写一个app或者让它具有社交账号api。
- 路线规划：显然用agent系统来实现，本项目初期就是用一个支持agent和RAG系统的LlamaIndex作为基础的开源框架。不过对于agent，初期我可能会用一个成熟的ai api来参与并和它对话，强调它的职责；后期也可以保留，一个是用于审查，另一个可以作为架空设定里工具的一环和星伞参与交互。
### 交互前端
- 目标：让AI可以在网站端被访问。
- 路线规划：接着改我的小网站，日后也会放在这个页面的。
### 多模态感知（长期目标）
- 目标：像neuro那样搞一个会动会做动作的皮套、能够识别图像和音乐，让它成为一个更加立体的形象。
- 路线规划：多模态后期再做，至于皮套可以先约稿，有了以后再通过agent来实现像虚拟主播一样动，感觉又是一个深坑。
### 情感化模块（长期目标）
- 目标：为了让AI更多地获得情绪反馈，可能会考虑加一个人工的情感反馈模块来模拟情绪感知。
- 路线规划：其实就是多一个学习参数的事情，还是比较好做的，难点还是在收集数据集这块。
